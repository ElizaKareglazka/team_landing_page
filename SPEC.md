# X5 Tech — Сайт команды разработки

## Спецификация проекта (v1.0)

---

## 1. Общее описание

Одностраничный сайт-визитка команды разработки X5 Tech. Выполняет функцию электронной визитки: рассказывает о команде, её участниках, проектах, процессе работы и текущем roadmap. Даёт внешним и внутренним заказчикам понимание, чем занимается команда и с какой задачей к ней можно обратиться.

### Целевая аудитория
- Внутренние заказчики (бизнес-подразделения X5)
- Потенциальные сотрудники
- Коллеги из смежных команд

### Хостинг
- GitHub Pages (статический хостинг)
- Без серверной части (backend)
- Все данные хранятся в `data.json`

### Технологический стек
- HTML5, CSS3, JavaScript (vanilla, без фреймворков)
- Шрифты: Manrope (основной), JetBrains Mono (моноширинный, акцентный)
- Никаких зависимостей, сборщиков или npm

### Типографика
- **Заголовки**: Manrope, 700–800, letter-spacing: -1…-2px
- **Основной текст**: Manrope, 400, 16px, line-height: 1.7
- **Моноширинный** (теги, метки, код): JetBrains Mono, 11–13px, letter-spacing: 0.5–3px
- **Секционные метки**: JetBrains Mono, 11px, uppercase, зелёный цвет, с декоративной линией слева

### Визуальные эффекты
- Grain-текстура поверх всей страницы (SVG noise, opacity: 0.03)
- Scroll-reveal анимации (fade-in + translate снизу)
- Hover-эффекты на карточках (подъём, зелёный бордер, зелёная полоса сверху)
- Зелёное свечение (radial-gradient) в hero-секции
- Кастомный скроллбар (тёмный, зелёный при hover)

## 3. Структура страницы (секции)

### 3.1. Header (фиксированный)
- **Левая часть**: логотип «X5 Tech» (текстовая заглушка, позже — изображение из `assets/logo/`)
- **Правая часть**: навигация — ссылки на секции (О нас, Команда, Проекты, Roadmap, Процесс, FAQ)
- Фон: полупрозрачный с backdrop-filter: blur
- Активная секция подсвечивается зелёным при скролле

### 3.2. Hero
- Тег-метка: «Команда разработки» (моноширинный)
- Заголовок: крупный, с акцентным словом зелёного цвета + подчёркивание
- Подзаголовок: 1–2 предложения о команде
- Метрики: 3 числовых показателя (кол-во разработчиков, проектов, лет опыта) — данные из `data.json` поле `meta`
- CTA-кнопки: «Связаться с нами» (primary, ссылка на почту) + «Смотреть проекты» (ghost, якорь на секцию)

### 3.3. О команде
- Метка секции + заголовок + описание
- 4 карточки направлений (веб-разработка, бэкенд, данные, DevOps)
- Каждая карточка: иконка (символ), заголовок, краткое описание
- Данные: из `data.json` поле `directions`

### 3.4. Стек технологий
- Бесконечная бегущая строка (CSS animation, дублирование контента)
- Каждый элемент: зелёная точка + название технологии (моноширинный)
- Hover: зелёный бордер + зелёный текст
- Маска по краям (gradient mask для плавного исчезновения)
- Данные: из `data.json` поле `techStack` (массив строк)

### 3.5. Участники команды
- Сетка карточек: `grid-template-columns: repeat(auto-fill, minmax(180px, 1fr))`
- Карточка сотрудника:
  - Фото (72×72px, круглое, grayscale → цвет при hover)
  - Имя (15px, bold)
  - Роль (12px, моноширинный, серый)
  - Подсказка «Подробнее →» (появляется при hover)
- **Клик по карточке** → модальное окно с подробной информацией

#### Модальное окно сотрудника
- Фото (80×80px, круглое, зелёный бордер)
- Имя + роль (зелёный, моноширинный)
- Секция «О сотруднике»: текстовое описание
- Секция «Навыки»: теги (зелёный фон, моноширинный)
- Секция «Контакты»: кнопки-ссылки (email, Telegram)
- Закрытие: кнопка ✕, клик по overlay, клавиша Escape

#### Fallback для фото
- Если путь к фото пустой или файл не загружается — показывается SVG-аватар с инициалами сотрудника
- Инициалы генерируются автоматически из поля `name`
- Фон аватара: `--green-dim`, текст: `--green`

#### Данные сотрудника в `data.json`

  "name": "Алексей Петров",

    { "type": "email", "value": "a.petrov@x5.tech" },

### 3.6. Проекты / Кейсы
- Сетка: `grid-template-columns: repeat(auto-fill, minmax(320px, 1fr))`
- Карточка проекта:
  - Обложка (160px высота): изображение из `cover` или SVG-паттерн (fallback)
  - Номер проекта (крупный, полупрозрачный, поверх обложки)
  - Тег категории (моноширинный, зелёный, uppercase)
  - Заголовок + описание
  - Теги стека технологий
- **Клик по карточке** → раскрывается блок с деталями:
  - «Задача» — что нужно было сделать
  - «Результат» — что получилось (метрики)

#### Fallback для обложки
- Если поле `cover` пустое — рендерится SVG-паттерн
- Паттерны различаются по категории проекта (сетка, линии, круги и т.д.)

### 3.7. Roadmap (НОВАЯ СЕКЦИЯ)
Диаграмма Ганта, отображающая текущие и планируемые проекты команды.

#### Визуальное представление
- **Горизонтальная ось**: месяцы (отображаем окно 6–12 месяцев)
- **Вертикальная ось**: названия проектов
- **Полосы проектов**: горизонтальные бары от даты старта до даты окончания
- **Линия «Сегодня»**: вертикальная пунктирная линия зелёного цвета с меткой
- **Статусы** (цветовая кодировка):
  - `done` — приглушённый зелёный (#00e85e55), сплошная заливка
  - `in_progress` — яркий зелёный (#00e85e), с пульсирующей анимацией
  - `planned` — серый (#333), пунктирная рамка

- Горизонтальная прокрутка если проектов/месяцев много
- Адаптив: на мобильных — горизонтальный скролл с подсказкой свайпа

### 3.8. Как мы работаем (процесс)
- Горизонтальный таймлайн из 5 шагов
- Каждый шаг: номер (моноширинный, зелёный) + линия с точкой + заголовок + описание
- Последний шаг: зелёная точка на конце линии (финальная)
- Данные: из `data.json` поле `process`
- Адаптив: на мобильных — вертикальный layout

### 3.9. FAQ / С чем к нам прийти
- Аккордеон: 4–6 вопросов

- При клике раскрывается текст ответа (анимация max-height)
- Одновременно открыт только один элемент
- Данные: из `data.json` поле `faq`

### 3.10. Footer
- Логотип (текст) + копирайт
- Ссылки: email (mailto:), Telegram (внешняя ссылка)
- Данные контактов: из `data.json` поле `contacts`

## 4. Админ-режим

### 4.1. Вход
- Активация: параметр `?admin=1` в URL **или** тройной клик по логотипу в header
- Появляется модалка с полем ввода пароля
- Пароль проверяется через SHA-256 хэш (хранится в `js/admin.js`)
- При успешном входе: `sessionStorage.setItem('admin', 'true')`
- Режим сохраняется до закрытия вкладки

### 4.2. Плавающая админ-панель
- Появляется поверх сайта (fixed, top)
- Содержит:
  - Индикатор «Режим администратора»
  - Кнопка «Скачать data.json» — экспортирует текущее состояние данных
  - Кнопка «Импортировать data.json» — загружает файл и применяет данные
  - Кнопка «Выйти» — деактивирует режим

### 4.3. Редактирование секций
При активном админ-режиме у каждой редактируемой секции появляются кнопки:

#### Команда
- Кнопка «+ Добавить сотрудника» после последней карточки
- На каждой карточке: кнопки «Редактировать» и «Удалить»
- Форма добавления/редактирования (модалка):
  - Имя (text)
  - Роль (text)
  - Путь к фото (text, с подсказкой формата)
  - Описание (textarea)
  - Навыки (text, через запятую)
  - Email (text)
  - Telegram (text)

#### Проекты
- Кнопка «+ Добавить проект» после последней карточки
- На каждой карточке: кнопки «Редактировать» и «Удалить»
- Форма добавления/редактирования (модалка):
  - Категория/тег (text)
  - Название (text)
  - Краткое описание (textarea)
  - Путь к обложке (text)
  - Стек технологий (text, через запятую)
  - Задача (textarea)
  - Результат (textarea)

#### Roadmap
- Кнопка «+ Добавить проект в roadmap»
- Таблица-форма со строками для каждого элемента roadmap:
  - Название проекта (text)
  - Дата начала (date, формат YYYY-MM)
  - Дата окончания (date, формат YYYY-MM)
  - Статус (select: planned / in_progress / done)
  - Ответственный (select из списка сотрудников)
  - Описание (text)
  - Кнопка удаления строки

#### FAQ
- Кнопка «+ Добавить вопрос»
- На каждом элементе: кнопки «Редактировать» и «Удалить»
- Форма: вопрос (text) + ответ (textarea)

#### Метаданные (hero)
- Редактирование метрик, слогана, подзаголовка
- Редактирование контактов (email, Telegram)

### 4.4. Процесс сохранения
1. Все изменения применяются к объекту `window.siteData` в памяти
2. Сайт мгновенно перерендеривается с новыми данными
3. Для сохранения: кнопка «Скачать data.json» формирует и скачивает файл
4. Пользователь коммитит файл в репозиторий
5. GitHub Pages обновляется автоматически

## 7. Требования к изображениям

### Фото сотрудников
- Формат: JPG, PNG или WebP
- Размер: рекомендуется 200×200px (квадрат)
- Отображение: 72×72px в карточке, 80×80px в модалке
- Имя файла: латиница, lowercase, без пробелов (например `petrov.jpg`)
- Fallback: SVG-аватар с инициалами

### Обложки проектов
- Формат: JPG, PNG или WebP
- Размер: рекомендуется 640×360px (16:9)
- Отображение: ширина по карточке, высота 160px, object-fit: cover
- Имя файла: slug проекта (например `api-gateway.png`)
- Fallback: SVG-паттерн по типу проекта

### Логотип
- Формат: SVG (предпочтительно) или PNG с прозрачным фоном
- Размер: высота ~32px в header

## 8. Адаптивность

- **Desktop** (>1024px): полная сетка, горизонтальный таймлайн, Ганта со скроллом
- **Tablet** (768–1024px): сетки адаптируются, карточки по 2–3 в ряд
- **Mobile** (<768px):
  - Навигация: уменьшенный размер шрифта, возможно бургер-меню (v2)
  - Карточки: по 2 в ряд (min 150px)
  - Проекты: в одну колонку
  - Таймлайн процесса: вертикальный
  - Ганта: горизонтальный скролл с подсказкой
